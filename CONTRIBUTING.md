# Guía de Contribución

## ¿Cómo contribuir?

1. Haz un fork del repositorio
2. Crea un branch para tu feature: `git checkout -b mi-feature`
3. Haz commit de tus cambios: `git commit -m "feat: mi nueva feature"`
4. Push al branch: `git push origin mi-feature`
5. Abre un Pull Request

## Convenciones de commits

Usamos [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` nueva funcionalidad
- `fix:` corrección de bug
- `docs:` cambios en documentación
- `refactor:` refactorización sin cambio funcional
- `test:` agregar o modificar tests

## Estilo de código

- Seguir PEP 8
- Type hints en funciones públicas
- Docstrings en módulos y funciones principales
